⚖ 비교표
매체 특성 — 텍스트 기반 마크업. 별도 뷰어 없이 브라우저만으로 재현. W3C 표준으로 1990년대 HTML도 현재 브라우저에서 정상 작동 — 가장 긴 하위 호환 이력.

포함 내용 — 대통령 발언·서한·담화문 본문 + 메타데이터(날짜·분류·등록자). HWP 에디터 생성 HTML을 e지원 템플릿이 래핑한 구조.

구조 — 상단 .ct_01: 제목·메타데이터 / 하단 BODY: 단락(P 태그). 인라인 CSS. 굴림·돋움체 선언.

이관 유의사항
① charset=euc-kr 선언 — EUC-KR로 읽어야 함. UTF-8로 열면 한국어 깨짐
② iconv -f EUC-KR -t UTF-8 변환 후 U+FFFD 0개 검증 필수
③ charset 메타태그도 UTF-8로 교체해야 브라우저 정상 렌더링
④ 외부 리소스 없음 — 단일 파일 자급자족 구조
매체 특성 — 국가기록원 RMS 장기보존 표준. EncapsulatedObject 구조 — 메타데이터·본문·첨부(Base64)·전자서명을 단일 XML에 캡슐화. SHA-256 + GPKI 이중 서명으로 무결성 보증.

포함 내용
① 객체 메타데이터: 기관·부서·생산자·날짜·보존기간·공개유형
② 문서관리카드(mmcContent): 보고 취지·문서구분·결재자
③ 결재경로(dmcPath): 담당자·날짜·의견·처리결과 — 의사결정 계층과 환류 과정의 핵심
④ 첨부파일(HWP·PDF·PPT): Base64 내장 — 별도 디코딩 필요
⑤ 전자서명: 기록관리비서관 + 서버 이중 서명

결재경로의 의의 — e지원의 핵심. 비서관→수석→대통령(참조)→비서실장→수보회의→시행 계층이 타임스탬프와 함께 기록. 반환·재상신 이력 보존. 의사결정 흐름 재구성의 유일한 근거.

이관 유의사항
① UTF-8 — 인코딩 문제 없음 ② DocumentData: Base64 첨부 — 별도 디코딩 필요
③ 전자서명: GPKI 의존 — 현재는 서명 값만 보존 ④ KDN 속성이 의미 표지 — tag명보다 KDN 우선 참조
